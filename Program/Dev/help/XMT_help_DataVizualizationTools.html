This help file contains a description of the module Data Visualisation, which can be used to plot mapping data in histograms, binary plots, ternary diagrams and as a RGB composite image.

The module Data Visualisation has access to the data selected in the Primary Menu of XMapTools when it is opened. To change the type of data, close the module and re-open it after selecting different data.

Visualisation modes: histogram, binary, RGB and ternary

The type of diagram to be generated can be selected in the section Plot. The following modes are available:

  • Histogram, requires a single variable X; an histogram is generated and plotted in the main figure (Fig. 1)
  • Binary plot, requires two variables X and Y; a binary plot is generated and plotted in the main figure, two histograms, one for each dimension and a density image (not shown if the low ressource mode is active) are plotted as additional figures (Fig. 2)
  • RGB, requires three variables X, Y and Z; a RGB composite image is generated and plotted in the main figure, a legend is plotted in an additional figure (Fig. 3)
  • Ternary diagram, requires three variables X, Y and Z:; a ternary plot is generated and plotted in the main figure, three histograms, one for each dimension and a density image (not shown if the low ressource mode is active) are plotted as additional figures (Fig. 4)

By default, XMapTools generates a binary diagram when the module is opened if at least two variables (maps) are available.

Variable definition

On the left side of the section Plot, it is possible to select the maps to be displayed. These can be elements, groups of elements or any other variable (i.e., structural formula)

Regardless of the type of the plot, variables are selected in Variable definitions, either using the dropdown menus listing all maps available, or using any combinaison of them.

Standard MATLAB code format is used to combine existing maps and create new variables to be plotted. Use arithmetic operation defined by the rules of linear algebra or array arithmetics. The operation can contain empty spaces between variables digits and operators. Array operators (with the period character ".") must be used if maps are divided or multiplied by each other. Note that element-wise multiplication or division should be used ".*" or "./" in order to multiply or divide the two arrays element by element.

Data sampling (Explore section)

A basic identify pixel tool is available only for binary and ternary plots. Use this fuction to create polygons identifying the location of points in the map.

When a polygon is drawn, the pixel locations of all points within the region-of-interest ss displayed on the map located at the bottom right. The shape can be edited and the plot is updated accordingly. An identical polygon is plotted on the density map if the low ressource mode is off.

The button Polygon activates a cross cursor and you can select vertices of a polygon around any region-of-interest in the main plot. Right click to close the polygon.

It is possible to create several polygons, by clicking on the button to active the cursor. The distribution on the map (displayed on the right) updates authomatically.

Masks

Box and dropdown menu Masks

By selecting the box Masks the data displayed on the binary or ternary plots are colored according to the mask file displayed on the dropdown menu (Figure 2).

Figure 5: Intensity data colored according to minerals retrieved from the selected mask file.

Low ressource mode

This mode is optimal for computers with limited memory and his activated by default when the Data Visualisation Module opens. When this mode is activated, only 40% of the data are plotted (random selection).

For binary and ternary plots, density maps calculated and displayed only if the 'Low ressource mode' is off (Fig. 2 & 4).

Statistics

Statistics are reported in the section statistics and they are updated every time the plot is changed.
